# Feature Specification: 知识图谱独立页面与统计页面精简

**Feature Branch**: `feat/011-knowledge-graph-split`

**Created**: 2026-05-25

**Status**: Draft

**Input**: 页面结构调整：将 Stats 页面的知识图谱相关区块拆分为独立页面，Stats 页面保留纯统计数据。

> ⚠️ **Constitution Note**: 本 spec 新增第六个导航页面「知识图谱」，与宪法 Principle V（固定五页导航）存在冲突。实施前需通过 `/speckit-constitution` 完成宪法修订，将导航页数更新为 6 页。

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 在独立页面浏览 Mira 的知识图谱 (Priority: P1)

用户希望了解 Mira 的知识状态——它当前识别的项目、实体和最近的决策推断。这些信息原先埋藏在统计页面底部，容易被忽视。现在用户可以直接通过导航菜单进入「知识图谱」页面，获得清晰的全貌。

**Why this priority**: 知识图谱是 Mira 作为智能助理的核心价值呈现，应有独立的信息架构地位，而非统计数据的附属内容。

**Independent Test**: 在导航菜单中点击「知识图谱」，验证：页面正常打开；显示 Mira 最近的思考摘要；展示项目列表（卡片形式）；展示识别出的实体（Tag 云）；展示最近 5 条决策。

**Acceptance Scenarios**:

1. **Given** 用户进入应用，**When** 查看导航菜单，**Then** 菜单中存在「知识图谱」条目，位于「统计」之后，使用节点/脑图类图标，点击可进入对应页面。
2. **Given** 用户点击进入「知识图谱」页面，**When** 页面加载完成，**Then** 页面顶部显示「Mira 最近在想什么」区域，以轻量文字段落形式呈现最近的 Lint 思考摘要，内容为中文。
3. **Given** 用户查看「知识图谱」页面中部，**When** 存在已识别的项目，**Then** 项目以卡片形式展示，每张卡片显示：项目名称、关联任务数量、项目状态；卡片间均匀排列。
4. **Given** 用户查看「知识图谱」页面下半部，**When** 存在已识别的实体，**Then** 左侧区域以 Tag 云形式展示识别到的人物、工具和概念实体，不同类别的 Tag 有视觉区分。
5. **Given** 用户查看「知识图谱」页面下半部右侧，**When** 存在历史决策，**Then** 显示最近 5 条决策记录，以单行列表形式展示，按时间倒序排列。
6. **Given** 知识库尚未有任何数据（全新安装），**When** 用户进入「知识图谱」页面，**Then** 各区域显示中文空态提示，引导用户了解如何生成知识图谱数据（如「Mira 还没有开始整理知识，继续记录后会自动生成」）。

---

### User Story 2 — 在精简后的统计页面查看纯数据统计 (Priority: P2)

用户需要快速了解自己的工作产出概况（待办完成情况、笔记积累、项目活跃度）。精简后的统计页面专注于数字摘要和趋势图，不再混入知识图谱内容，信息更聚焦。

**Why this priority**: 统计页面承担「量化工作产出」的职责；移除知识图谱区块后，应以统计卡片填满页面，避免布局空洞。

**Independent Test**: 进入统计页面，验证：无知识图谱/Schema 相关区块；顶部展示 4 个统计卡片一行；存在趋势图表或统计摘要；页面无大片空白。

**Acceptance Scenarios**:

1. **Given** 用户点击导航进入「统计」页面，**When** 页面加载完成，**Then** 页面标题显示「统计」，不存在任何知识图谱、Schema、Lint 摘要相关的区块或入口。
2. **Given** 用户查看统计页面顶部，**When** 有历史数据，**Then** 顶部一行展示 4 个统计卡片：本周完成待办数、本周新增待办数、笔记总数、活跃项目数；数字使用品牌强调色突出显示。
3. **Given** 用户查看统计页面，**When** 存在统计数据，**Then** 页面还展示：本月完成待办数、本月新增待办数、本周新增笔记数、知识库最后更新时间等补充指标。
4. **Given** 用户查看统计页面图表区域，**When** 存在近 7 天的待办记录，**Then** 显示近 7 天待办完成趋势的折线图，坐标轴标签为中文日期格式。
5. **Given** 统计页面移除了知识图谱区块，**When** 用户查看页面整体布局，**Then** 统计卡片和图表区域填满页面，不存在超过 1/4 页面高度的大片空白区域。
6. **Given** 应用全新安装无历史数据，**When** 用户进入统计页面，**Then** 各统计数字显示 0 或「暂无数据」，折线图区域显示中文空态提示，页面布局整洁。

---

### Edge Cases

- 知识图谱数据加载失败（网络或系统错误）：页面各区域应独立降级，显示「数据加载失败，请稍后重试」，不影响其他区域渲染。
- 项目列表超过 10 个：卡片网格应支持滚动或翻页，不截断数据。
- 实体 Tag 数量超过 50 个：Tag 云应有最大显示数量限制，超出部分可折叠或省略，不破坏布局。
- 统计页面在宪法修订完成前不应出现任何「知识图谱」入口或引导链接，保持页面独立性。

## Requirements *(mandatory)*

### Functional Requirements

**F06 — 知识图谱独立页面**

- **FR-001**: 应用导航菜单 MUST 新增「知识图谱」条目，位于「统计」之后，使用节点/脑图类图标。
- **FR-002**: 「知识图谱」页面 MUST 可通过菜单导航直接访问，路径独立。
- **FR-003**: 「知识图谱」页面顶部 MUST 展示「Mira 最近在想什么」区域，内容为最近 Lint 思考摘要的简短中文描述。
- **FR-004**: 「知识图谱」页面中部 MUST 以卡片网格形式展示已识别的项目列表，每张卡片 MUST 包含：项目名称、关联任务数量、项目状态；项目状态 MUST 为三值枚举：「进行中」「已暂停」「已完成」，各状态 MUST 有视觉区分（如颜色 Badge）。
- **FR-005**: 「知识图谱」页面下部左侧 MUST 以 Tag 云形式展示已识别的实体，实体类型（人物/工具/概念）MUST 有视觉区分（如颜色或图标前缀）。
- **FR-006**: 「知识图谱」页面下部右侧 MUST 展示最近 5 条决策记录，以单行列表形式按时间倒序排列。
- **FR-007**: 「知识图谱」页面所有区域 MUST 在无数据时显示中文空态提示，引导用户理解数据来源。
- **FR-008**: 「知识图谱」页面数据来源为后台已有的知识图谱查询能力，不新增后台存储逻辑。
- **FR-008a**: 「知识图谱」页面数据 MUST 在本次运行会话内缓存；页面 MUST 提供「刷新」按钮，点击后重新请求最新数据并更新展示内容。首次访问该页面时自动加载一次数据。

**F07 — Stats 页面精简**

- **FR-009**: 统计页面 MUST 移除所有知识图谱、Schema、Lint 摘要相关区块，不保留任何跳转到知识图谱的入口。
- **FR-010**: 统计页面顶部 MUST 展示第一行 4 个统计卡片：本周完成待办数、本周新增待办数、笔记总数、活跃项目数。
- **FR-011**: 统计页面 MUST 展示补充统计指标：本月完成待办数、本月新增待办数、本周新增笔记数、知识库最后更新时间。
- **FR-012**: 统计页面 MUST 展示近 7 天待办完成趋势折线图（若系统已有相关图表组件则保留，否则新增）；折线图坐标轴标签 MUST 为中文日期格式。
- **FR-013**: 统计页面移除区块后 MUST 用统计卡片和图表填满布局，不允许存在超过页面高度 1/4 的大片空白区域。
- **FR-014**: 统计页面所有文案和数字标签 MUST 使用中文。
- **FR-015**: 统计页面无数据时 MUST 显示中文空态提示，各统计数字显示 0，折线图显示空态说明。

### Key Entities

- **WikiSchema（知识图谱数据）**: 包含 Mira 识别到的项目列表、实体集合（人物/工具/概念）、决策历史和 Lint 摘要；由后台持续更新，前端只读展示。
- **Project（项目）**: 知识图谱中识别的工作项目，具有名称、关联任务数和状态属性；状态为三值枚举：「进行中」「已暂停」「已完成」。
- **Entity（实体）**: 从用户记录中提取的人物、工具或概念，以 Tag 形式展示。
- **Decision（决策）**: Mira 推断出的用户历史决策记录，包含描述和时间戳。
- **StatsSnapshot（统计快照）**: 汇总用户在一定时间范围内的工作量数字，包括待办完成数、新增数、笔记数等。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 导航菜单可见「知识图谱」条目，点击后在 1 秒内打开页面并开始渲染内容。
- **SC-002**: 「知识图谱」页面在有数据时，4 个内容区域（摘要、项目列表、实体云、决策列表）全部正确展示数据，无区域空白或错误。
- **SC-003**: 统计页面在视觉审查中无任何知识图谱或 Schema 相关区块，验证通过率 100%。
- **SC-004**: 统计页面顶部 4 张统计卡片在有数据时均显示非零数字，折线图在有历史数据时正确渲染 7 天趋势。
- **SC-005**: 统计页面整体布局紧凑，无大片空白（目视检查通过）。
- **SC-006**: 两个页面的所有用户可见文案均为中文，无英文残留。

## Assumptions

- 「知识图谱」页面的数据来源为系统已有的 `get_wiki_schema()` 后台能力，本 spec 不新增后台存储或 AI 处理逻辑。
- 「知识图谱」页面是只读的信息展示页，用户不能在此页面编辑或删除知识图谱数据。
- 统计数据（待办数、笔记数等）来源于系统现有数据库，全部 8 项统计指标（本周/本月完成待办数、本周/本月新增待办数、笔记总数、本周新增笔记数、活跃项目数、知识库最后更新时间）均可从现有数据和查询能力派生，**无需新增 Rust command**；若某项统计数据暂时不可用，该指标卡片显示「暂无数据」。
- 「近 7 天趋势折线图」：若系统已有图表组件则复用，若无则新增简单折线图；不引入新的图表库。
- 宪法 Principle V 已通过 `/speckit-constitution` 修订至 v3.0.0，导航页数更新为 6，冲突已解除。
- 知识图谱原有组件代码不删除，仅从 Stats 页面移动到新页面。

## Clarifications

### Session 2026-05-25

- Q: 统计页面的 8 项指标（本周/本月计数等）是否需要新增后台查询命令？→ A: 全部可从现有数据和查询能力派生，无需新增 Rust command。
- Q: 知识图谱页面数据刷新策略？→ A: 会话内缓存 + 手动「刷新」按钮；首次访问自动加载一次。
- Q: 知识图谱项目卡片的「项目状态」可取哪些值？→ A: 三值枚举：进行中 / 已暂停 / 已完成，各状态有视觉区分。
