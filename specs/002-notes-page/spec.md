# Feature Specification: Notes 页面

**Feature Branch**: `002-notes-page`

**Created**: 2026-05-25

**Status**: Draft

**Input**: 新增「Notes」页面，用于日常工作随手笔记、会议记录、思考片段。
Markdown 编辑，自动保存，LLM Wiki 后台处理分类，无需用户打标签。

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 浏览笔记列表 (Priority: P1)

用户切换到「笔记」标签页，看到所有已有笔记的列表，按创建时间降序排列，
每条显示前两行预览和时间戳。列表为空时显示引导文案。

**Why this priority**: 这是页面的入口态。无论其他功能是否完成，列表展示
可以独立交付并验证。

**Independent Test**: 有笔记时查看列表，确认降序排列、每条有预览和时间戳；
无笔记时确认显示空态文案。

**Acceptance Scenarios**:

1. **Given** 有 5 条笔记，**When** 用户进入「笔记」页，**Then** 列表按创建时间
   降序显示，每条展示前两行文字预览和可读的时间戳（如「3 分钟前」或「05-25 14:30」）。
2. **Given** 没有任何笔记，**When** 用户进入「笔记」页，**Then** 显示空态文案
   「还没有笔记，记录今天的第一个想法吧」，无列表项。
3. **Given** 一条笔记内容超过两行，**When** 显示在列表中，**Then** 仅展示前两行
   预览，超出部分截断（不换行堆叠）。

---

### User Story 2 — 新建并编辑笔记 (Priority: P1)

用户点击「+ 新建笔记」进入全屏编辑模式，支持 Markdown 语法，可实时预览
（左侧编辑、右侧预览，或单面板切换模式）。内容自动保存，无需手动点击保存。

**Why this priority**: 这是页面的核心功能，与浏览列表同等重要，共同构成最小
可用版本。

**Independent Test**: 点击新建，输入 Markdown 内容，稍等片刻后返回列表，
确认笔记已出现在列表顶部，内容与输入一致，全程无需手动保存。

**Acceptance Scenarios**:

1. **Given** 用户在笔记列表，**When** 点击「+ 新建笔记」，**Then** 进入全屏
   编辑器，编辑区显示 placeholder「开始记录…」，焦点在编辑区。
2. **Given** 编辑器打开，**When** 用户输入 Markdown（如 `# 标题` 和正文），
   **Then** 预览区实时渲染为格式化结果，停顿约 1 秒后系统自动保存。
3. **Given** 用户已输入内容且自动保存已触发，**When** 用户退出编辑器返回列表，
   **Then** 该笔记出现在列表顶部，预览显示前两行内容。
4. **Given** 编辑器打开但内容为空，**When** 用户退出编辑器，**Then** 不创建
   空笔记，列表不新增条目。
5. **Given** 用户已有一条笔记，**When** 在列表点击该笔记，**Then** 重新进入
   编辑器并显示已有内容，可继续编辑。

---

### User Story 3 — 删除笔记 (Priority: P2)

用户通过长按（或右键菜单）触发删除选项，系统弹出二次确认对话框，确认后删除。

**Why this priority**: 删除是补充能力，在浏览和新建之后交付，不影响 MVP。

**Independent Test**: 长按或右键一条笔记，选择删除，在二次确认弹窗中点确认，
确认笔记从列表中消失；在弹窗中点取消，确认笔记保留。

**Acceptance Scenarios**:

1. **Given** 列表中有一条笔记，**When** 用户长按该条目（或右键），
   **Then** 出现包含「删除」选项的上下文菜单。
2. **Given** 用户选择「删除」，**When** 确认弹窗显示，**Then** 弹窗文案为
   「确认删除这条笔记？」，提供确认和取消两个按钮。
3. **Given** 确认弹窗显示，**When** 用户点击确认，**Then** 笔记从列表中移除。
4. **Given** 确认弹窗显示，**When** 用户点击取消，**Then** 弹窗关闭，笔记保留。

---

### User Story 4 — Copilot 澄清提问条 (Priority: P3)

自动保存触发后，若 LLM Wiki 后端需要澄清，在编辑器底部出现一条提问条
（前缀「Mira：」，不遮挡编辑区）。用户可文字回答后提交，或点「跳过」。

**Why this priority**: 增强 AI 知识库质量，与主功能解耦，可独立交付。

**Independent Test**: 模拟触发 Copilot 澄清，确认提问条出现在编辑器底部、
不遮挡编辑区，回答或跳过后消失。

**Acceptance Scenarios**:

1. **Given** 自动保存触发且后端返回澄清请求，**When** 编辑器界面激活，
   **Then** 编辑器底部出现提问条，前缀「Mira：」，后跟问题内容，
   提供输入框和「跳过」按钮。
2. **Given** 提问条可见，**When** 用户输入回答并提交，**Then** 提问条消失，
   回答发送给后端，编辑器保持可用。
3. **Given** 提问条可见，**When** 用户点击「跳过」，**Then** 提问条消失，
   不发送任何内容，编辑器保持可用。
4. **Given** 提问条可见，**When** 用户继续在编辑区输入，**Then** 提问条不影响
   编辑操作，不抢占焦点。

---

### Edge Cases

- 编辑器内容极长（超过 10000 字符）时，自动保存不应阻塞编辑体验。
- 快速连续输入期间，自动保存以防抖策略触发（停止输入后约 1 秒），不因频繁
  保存产生感知延迟。
- 多条 Copilot 澄清同时触发时，每次只展示一条提问条，按队列顺序处理。
- 编辑器处于空内容状态时，不触发自动保存和 Copilot 查询。
- 应用离线时，自动保存操作应给出中文错误提示，内容不丢失。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 「笔记」标签页 MUST 为五标签导航中的第二个页面（Todo 之后）。
- **FR-002**: 笔记列表 MUST 按创建时间降序排列，每条显示前两行预览文字和时间戳。
- **FR-003**: 列表为空时 MUST 显示空态文案「还没有笔记，记录今天的第一个想法吧」。
- **FR-004**: 页面右上角 MUST 提供「+ 新建笔记」按钮，点击进入全屏编辑器。
- **FR-005**: 编辑器 MUST 支持 Markdown 语法，并提供实时预览能力（至少支持
  左写右看或单面板切换两种预览模式之一）。
- **FR-006**: 编辑器 MUST 显示 placeholder 文案「开始记录…」（内容为空时）。
- **FR-007**: 系统 MUST 在用户停止输入约 1 秒后自动保存笔记，无需用户手动触发
  保存动作。
- **FR-008**: 内容为空时，系统 MUST NOT 自动保存，也 MUST NOT 在列表中创建
  空笔记条目。
- **FR-009**: 点击列表中已有笔记 MUST 重新进入编辑器并加载该笔记内容。
- **FR-010**: 用户 MUST 能通过长按或右键上下文菜单触发删除操作。
- **FR-011**: 删除操作 MUST 弹出二次确认，文案「确认删除这条笔记？」，提供
  确认和取消。确认后删除，取消后保留。
- **FR-012**: 自动保存触发后，系统 MUST 向后端查询是否有 Copilot 澄清请求。
  若有，MUST 在编辑器底部显示提问条（前缀「Mira：」），不遮挡编辑区。
- **FR-013**: Copilot 提问条 MUST 提供文字输入框和「跳过」按钮；回答提交或
  跳过后提问条 MUST 消失。
- **FR-014**: 同一时刻最多显示一条 Copilot 提问条；多个请求按队列处理。
- **FR-015**: 页面 MUST NOT 提供笔记分类、标签、搜索或富文本编辑功能。

### Key Entities

- **Note（笔记）**: 用户创建的一条自由文本记录，包含内容（Markdown 原文）、
  创建时间、最后修改时间。无用户定义的分类或标签字段。
- **CopilotQuestion（澄清问题）**: 由后端 LLM Wiki 生成，包含问题文本，关联
  触发它的笔记；用户回答发送给后端，跳过则丢弃。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户从点击「+ 新建笔记」到可以开始输入，步骤不超过 1 次点击，
  感知响应时间不超过 1 秒。
- **SC-002**: 自动保存在用户停止输入后不超过 2 秒内完成，用户无需任何手动操作。
- **SC-003**: 编辑器关闭后返回列表，笔记内容与输入完全一致，不丢失任何已保存内容。
- **SC-004**: Copilot 提问条出现时，用户无需关闭提问条即可继续在编辑区输入，
  两者互不阻断。
- **SC-005**: 页面可见元素不包含任何分类、标签、搜索控件，视觉层保持「内容为主」。

## Assumptions

- 笔记数据通过 Tauri Rust 命令与本地 SQLite 存储，前端仅调用 invoke，不直接操作数据库。
- Markdown 预览在前端渲染，使用已存在或轻量级的 Markdown 解析库（已在项目依赖中或符合
  Principle X 的引入标准）。
- 自动保存防抖策略为停止输入后约 1 秒触发，具体阈值在实现阶段可微调，
  但不超过 2 秒（参照 SC-002）。
- Copilot 澄清问题由前端在自动保存后轮询或由后端推送，具体机制由后端规范决定；
  前端处理「有问题」和「无问题」两种状态。
- 「长按」手势适用于 macOS 触控板场景；「右键」适用于鼠标场景。两者均需支持，
  触发同一删除流程。
- 预览时间戳格式由实现阶段决定（如「X 分钟前」或绝对时间），
  只要对用户可读即符合规范。
- 本功能不变更导航框架，仅在现有五标签结构中替换「笔记」标签内容。
