# Feature Specification: 管理者专属团队功能页面

**Feature Branch**: `013-manager-team-pages`

**Created**: 2026-05-26

**Status**: Draft

**Input**: 为管理者新增两个专属页面：「团队统计」展示跨账号的团队数据汇总；
「团队问Mira」以所有成员数据为上下文进行跨团队 AI 问答。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 管理者查看团队本周工作全貌 (Priority: P1)

王五（管理者）登录后，进入「团队统计」页面，一眼看到三张总览卡片：本周团队完成了多少待办、
有多少活跃成员、产出了多少笔记。向下滚动查看每位成员的贡献明细，再向下看到各项目的进度条。
王五用 3 分钟完成了原本需要逐个账号登录才能掌握的团队全貌。

**Why this priority**: 团队统计是管理者权限的核心价值，直接解决了多账号数据分散无法汇总的问题。

**Independent Test**: 创建 3 个成员账号各自写入待办和笔记，管理者登录后查看团队统计页，
验证三张卡片的数字与各账号实际数据吻合。

**Acceptance Scenarios**:

1. **Given** 管理者已登录，**When** 进入「团队统计」页面，**Then** 页面顶部横排显示 3 张卡片：
   「本周团队完成待办总数」「团队活跃成员数」「团队笔记总数」，数值反映本周（周一至今）全账号数据。
2. **Given** 页面已加载，**When** 查看成员贡献列表，**Then** 每位成员显示一行：姓名、本周完成待办数、本周新增笔记数；按本周完成待办数倒序排列。
3. **Given** 成员已将任务关联到项目，**When** 查看项目进度汇总，**Then** 每个项目显示项目名称、
   参与成员头像/名字、本周完成任务数/总任务数、横向进度条。
4. **Given** 某成员本周无任何操作，**When** 管理者查看成员列表，**Then** 该成员仍显示在列表中，本周完成待办数和笔记数均显示为 0。
5. **Given** 所有成员本周均无数据，**When** 管理者查看统计页，**Then** 卡片数值显示 0，
   成员列表显示中文空状态提示「本周暂无活跃数据」。

---

### User Story 2 - 管理者向 Mira 询问跨团队问题 (Priority: P2)

王五想了解「本周各项目进展汇总」，进入「团队问Mira」页面，看到输入框上方的说明「基于全团队数据回答，
可询问跨成员的项目进展」，点击预设问题「本周各项目进展汇总」，Mira 综合所有成员的任务和笔记数据，
给出一份覆盖全团队的汇总回答。

**Why this priority**: 跨团队 AI 问答是团队管理的高价值功能，但依赖团队统计数据架构（P1）。

**Independent Test**: 创建 2 个成员账号各自完成若干任务，管理者登录后在团队问Mira 页
点击预设问题「团队本周完成了什么」，验证回答中包含来自两个账号的任务信息。

**Acceptance Scenarios**:

1. **Given** 管理者已登录，**When** 进入「团队问Mira」页面，**Then** 页面布局与个人「问Mira」页相同；
   输入框上方有灰色说明文字「基于全团队数据回答，可询问跨成员的项目进展」。
2. **Given** 管理者点击输入框，**When** 按下 Tab 键，**Then** 显示预设问题列表：
   「本周各项目进展汇总」「团队本周完成了什么」「哪个项目进展最慢」；点击任意预设问题自动填入输入框。
3. **Given** 管理者发送问题，**When** Mira 响应，**Then** 回答内容综合了所有成员账号的任务和笔记数据，
   而非仅管理者自己的数据。
4. **Given** AI key 未配置，**When** 管理者尝试发送消息，**Then** 显示中文提示「请先在设置中配置 API Key」，
   与个人问Mira页行为一致。
5. **Given** 团队无任何业务数据，**When** 管理者提问，**Then** Mira 回答「当前团队暂无数据」或
   给出合理的空数据回答，不报错。

---

### User Story 3 - 成员无法访问团队专属页面 (Priority: P3)

李四（成员）登录后，导航栏只显示 6 个页面（待办/笔记/问Mira/Mira的问题/统计/知识图谱），
「团队统计」和「团队问Mira」完全不可见，尝试直接访问对应路由也被拦截并跳回自己的待办页。

**Why this priority**: 权限隔离是安全前提；成员误入管理页会暴露全团队数据，需强制防护。

**Independent Test**: 成员账号登录后，检查导航栏条目数量，并尝试直接导航至团队统计路由，
验证被拒绝并重定向。

**Acceptance Scenarios**:

1. **Given** 成员账号已登录，**When** 查看导航栏，**Then** 导航栏只有 6 个条目，
   无「团队统计」和「团队问Mira」。
2. **Given** 成员尝试直接访问团队统计页路由，**When** 路由跳转，**Then** 被重定向至待办页，
   不渲染任何团队数据。

---

### Edge Cases

- 若某成员账号已被删除但其历史数据仍存在，成员贡献列表中该行仍然显示，姓名后附加「（已注销）」标记，其历史数据计入总览卡片的汇总数值中。
- 项目进度汇总中，若项目无关联任务，进度显示为 0/0 并隐藏进度条，不显示除以零错误。
- 团队问Mira 的上下文数据量若超过 AI 上下文窗口限制，tasks 和 notes 均按时间倒序排列后组装；超出窗口时从最旧的 notes 开始截断，tasks 最后截断；截断静默进行，不向用户报错。
- 「团队统计」和「团队问Mira」页面在没有 API Key 时，团队统计页正常展示数据（无 AI 依赖），
  团队问Mira 页显示配置提示。

## Clarifications

### Session 2026-05-26

- Q: 现有任务数据中是否存在「项目」字段用于项目维度汇总？ → A: 用任务现有的「分类」或「标签」字段作为项目维度，无分类/标签的任务归入「未分组」；无需新增字段。
- Q: 成员贡献列表中「最后活跃时间」如何定义？ → A: 不需要统计成员活跃时间，移除该字段。
- Q: 已删除账号的数据在团队统计中如何处理：排除还是保留标记？ → A: 保留已删除账号的行，标记为「已注销」，其历史数据仍计入总览卡片数值。
- Q: 「团队问Mira」的 AI 上下文是否应包含 tasks/notes 以外的数据（如 People）？ → A: 只包含 tasks 和 notes，不扩展至其他数据类型。
- Q: 团队数据超过 AI 上下文窗口时的截断策略？ → A: tasks 优先、notes 其次；两类数据均按时间倒序排列，超出窗口时从最旧的 notes 开始截断，tasks 最后截断。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 「团队统计」页面 MUST 仅在管理者 Session 下渲染，成员 Session 中该页面路由 MUST NOT 可访问。
- **FR-002**: 「团队问Mira」页面 MUST 仅在管理者 Session 下渲染，成员 Session 中该页面路由 MUST NOT 可访问。
- **FR-003**: 「团队统计」MUST 在页面顶部横排展示 3 张总览卡片：本周团队完成待办总数、团队活跃成员数（本周有记录的）、团队笔记总数（本周）。
- **FR-004**: 总览卡片数据 MUST 聚合当前设备上所有账号的数据，「本周」定义为本自然周周一 00:00 至当前时刻。
- **FR-005**: 「团队统计」MUST 在总览卡片下方展示成员贡献列表，每位成员一行，包含：姓名、本周完成待办数、本周新增笔记数；按本周完成待办数倒序排列，数量相同时按姓名字典序排列。
- **FR-006**: 「团队统计」MUST 在成员列表下方展示项目进度汇总，以任务的「分类/标签」字段作为项目维度分组；无分类/标签的任务归入「未分组」类别；每组显示：名称、参与成员（头像/名字）、本周完成任务数/总任务数、进度条。
- **FR-007**: 成员贡献列表 MUST 包含所有账号（含本周无活动的账号及已删除账号）；无活动账号计数显示 0；已删除账号姓名后 MUST 附加「（已注销）」标记，其历史数据仍计入总览卡片。
- **FR-008**: 「团队问Mira」页面布局 MUST 与现有「问Mira」页完全一致，复用同一布局组件。
- **FR-009**: 「团队问Mira」输入框上方 MUST 显示固定说明文字「基于全团队数据回答，可询问跨成员的项目进展」。
- **FR-010**: 「团队问Mira」输入框 MUST 支持 Tab 键调出预设问题列表：「本周各项目进展汇总」「团队本周完成了什么」「哪个项目进展最慢」，点击后自动填入输入框。
- **FR-011**: 「团队问Mira」发送消息时，AI 上下文 MUST 包含所有成员账号的 tasks 和 notes 数据，而非仅当前管理者账号数据。
- **FR-012**: 「团队问Mira」MUST 在无 API Key 配置时显示中文引导提示，与个人「问Mira」行为一致。
- **FR-013**: 两个新页面 MUST 在导航栏中位于现有「统计」和「知识图谱」页之间，顺序为：待办 → 笔记 → 问Mira → Mira的问题 → 统计 → 团队统计 → 团队问Mira → 知识图谱 → 账号管理（管理者视图共 9 页）。
- **FR-014**: 所有新增文案 MUST 为简体中文，包括标签、空状态提示、错误提示、卡片标题。

### Key Entities

- **TeamStats**: 团队统计聚合数据，包含 `total_tasks_done`（本周完成待办总数）、`active_members`（本周活跃成员数）、`total_notes`（本周笔记总数）、`member_stats`（成员贡献列表）、`project_stats`（项目进度列表）。
- **MemberStat**: 单个成员统计，包含 `account_id`、`name`、`tasks_done_this_week`、`notes_this_week`、`is_deleted`（布尔值，已删除账号为 true，UI 显示「已注销」标记）。
- **ProjectStat**: 单个分类/标签维度的统计，包含 `group_name`（分类/标签名，无分类任务显示「未分组」）、`members`（参与成员列表）、`tasks_done_this_week`、`total_tasks`、`progress_ratio`。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 管理者能在登录后 10 秒内查看到完整的团队统计页面，所有卡片和列表数据正确加载。
- **SC-002**: 团队统计数据 100% 准确 — 总览卡片数值与各成员账号实际记录的数据吻合，零容忍偏差。
- **SC-003**: 成员账号在任何操作路径下均无法看到「团队统计」或「团队问Mira」页面及其数据，任何暴露视为严重错误。
- **SC-004**: 管理者通过「团队问Mira」发出问题到收到回答的等待时间与个人「问Mira」页无显著差异（在相同网络和模型条件下）。
- **SC-005**: 「团队问Mira」的 AI 回答能正确引用来自不同成员账号的数据，覆盖率可在回答中验证。

## Assumptions

- 「本周」定义为自然周（周一 00:00 至周日 23:59），统计以设备本地时间为准。
- 项目维度使用任务现有的「分类/标签」字段（无需新增字段），无分类/标签的任务归入「未分组」类别。
- 成员头像使用姓名首字母占位圆形头像，不支持自定义头像上传。
- 「团队统计」不需要手动刷新按钮，进入页面时自动加载最新数据；不做实时推送。
- 「团队问Mira」的对话历史不在会话间持久化（与个人「问Mira」一致），关闭页面对话清空。
- 导航顺序从 7 页（管理者）改为 8 页，两个新页面插入在「统计」之后，「知识图谱」之前。
  这需要对应修订宪法 Principle V。
- 管理者自己的个人数据也包含在团队聚合中（管理者也是团队的一部分）。
- 不在本期实现：团队 LLM Wiki 自动聚合、成员间消息通知、权限细分。

## Out of Scope

- 团队 LLM Wiki 自动聚合
- 成员间消息通知
- 权限细分（本期只有 member 和 manager 两级）
- 团队数据的导出功能
- 历史统计（非本周数据的趋势图）
