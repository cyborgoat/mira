# Feature Specification: 前端菜单重构与五页导航

**Feature Branch**: `redesign/v2`

**Created**: 2026-05-25

**Status**: Draft

**Input**: 重构前端导航，调整为五个固定页面，用空白占位页验证路由与菜单正确性，确保未读 Badge 位置准确。

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 五页菜单导航（Priority: P1）

用户打开应用后，看到左侧菜单有且仅有五个导航项，按顺序排列：
「待办」「笔记」「问 Mira」「Mira 的问题」「统计」。
点击任意菜单项，应用切换到对应页面（即使页面内容尚为空白占位）。

**Why this priority**: 菜单结构是所有后续页面功能的基础框架。菜单顺序和路由不正确，任何页面都无法独立交付。

**Independent Test**: 启动应用，依次点击所有五个菜单项，确认每次切换后页面区域内容变化（哪怕仅是页面标题不同），且菜单高亮跟随当前页正确切换。

**Acceptance Scenarios**:

1. **Given** 应用启动完成，**When** 用户查看左侧菜单，**Then** 菜单中显示且仅显示五项：待办 / 笔记 / 问 Mira / Mira 的问题 / 统计，按此顺序排列，全部中文。
2. **Given** 用户当前在「待办」页，**When** 用户点击「笔记」，**Then** 主内容区域切换到笔记占位页，「笔记」菜单项高亮，其他项不高亮。
3. **Given** 应用启动，**When** 用户未进行任何操作，**Then** 默认激活页面为「待办」，该菜单项高亮。
4. **Given** 用户查看菜单，**When** 检查菜单中是否存在「设置」「LLM Wiki」等旧菜单项，**Then** 这两项不出现在菜单列表中。

---

### User Story 2 — Mira 的问题未读 Badge（Priority: P2）

用户在菜单栏的「Mira 的问题」菜单项上能看到一个数字 Badge，
反映当前未读 Lint 会话数量。当数量为零时 Badge 不显示或显示 0。

**Why this priority**: Badge 是「Mira 的问题」页面存在价值的视觉入口——没有 Badge，用户不知道有待处理的问题。此能力与菜单结构强耦合，需同步实现。

**Independent Test**: 在无 Lint 数据状态下启动应用，确认「Mira 的问题」菜单项不显示多余的 Badge 气泡（或显示 0 并符合设计）；通过 devtools 手动设置 unread_lint_count > 0，刷新后确认 Badge 显示正确数字。

**Acceptance Scenarios**:

1. **Given** 系统中未读 Lint 会话数量为 0，**When** 用户查看「Mira 的问题」菜单项，**Then** 该菜单项不显示 Badge 气泡（或显示表示零的视觉状态，不干扰菜单文字可读性）。
2. **Given** 系统中未读 Lint 会话数量为 3，**When** 用户查看菜单，**Then** 「Mira 的问题」菜单项旁边显示数字「3」的 Badge 气泡。
3. **Given** 用户点击「Mira 的问题」进入该页面，**When** 页面加载完成，**Then** 未读数应被清零，Badge 消失或变为 0。

---

### User Story 3 — 设置入口移至顶部栏（Priority: P3）

「设置」不再是左侧菜单的导航项，而是通过顶部栏的图标/按钮访问。
用户仍可正常进入设置页面，但入口位置改变。

**Why this priority**: 这是视觉布局调整，不影响核心功能，可在 P1/P2 完成后独立实现。

**Independent Test**: 查看左侧菜单确认无「设置」菜单项；点击顶部栏中的设置入口，确认能进入设置页面。

**Acceptance Scenarios**:

1. **Given** 用户打开应用，**When** 查看左侧导航菜单，**Then** 菜单中不存在「设置」条目。
2. **Given** 用户需要访问设置，**When** 点击顶部栏的设置图标或按钮，**Then** 主内容区域切换到设置页面。

---

### Edge Cases

- 应用刷新后，当前激活的页面应保持（通过 URL hash 或本地状态恢复）。
- 菜单文字较长时（如「Mira 的问题」），菜单项不应被截断，布局不破坏。
- Badge 数字超过 99 时，显示「99+」而非完整数字，以避免布局溢出。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 左侧导航菜单 MUST 包含且仅包含五个菜单项，按以下顺序排列：
  「待办」「笔记」「问 Mira」「Mira 的问题」「统计」，全部使用中文。
- **FR-002**: 每个菜单项点击后 MUST 切换主内容区域到对应页面（P1 阶段可为空白占位页面，显示页面中文标题即可）。
- **FR-003**: 当前激活的菜单项 MUST 以视觉高亮形式区别于其他菜单项，且与实际显示的页面保持同步。
- **FR-004**: 「Mira 的问题」菜单项 MUST 在菜单项旁显示未读数量 Badge；数量为 0 时 Badge MUST 隐藏或不显示。
- **FR-005**: Badge 数字超过 99 时 MUST 显示「99+」。
- **FR-006**: 用户进入「Mira 的问题」页面时 MUST 触发未读计数清零操作。
- **FR-007**: 「设置」MUST NOT 出现在左侧导航菜单中；设置页面 MUST 通过顶部栏入口访问。
- **FR-008**: 「LLM Wiki」MUST NOT 出现在任何用户可见导航区域。
- **FR-009**: 应用刷新后 MUST 恢复上次激活的页面（或默认「待办」）。
- **FR-010**: 所有菜单文字、页面标题、空态提示 MUST 为简体中文，无英文出现。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 五个菜单项全部为中文，顺序与规格一致，100% 可通过目测验证。
- **SC-002**: 点击任意菜单项后，主内容区域在 300ms 内完成切换（无加载延迟，因为是占位页）。
- **SC-003**: Badge 数字与系统中的未读 Lint 会话数量实时同步，误差为零。
- **SC-004**: 进入「Mira 的问题」页面后，未读 Badge 在同一次页面加载中消失，无需刷新。
- **SC-005**: 旧菜单项（设置、LLM Wiki）在任何导航区域均不可见，通过 UI 扫描 100% 确认。

## Assumptions

- 「待办」「笔记」「问 Mira」「Mira 的问题」「统计」五个占位页面内容仅需显示页面标题，无需完整功能。
- 未读数量初始值为 0（因为 Lint 数据尚未实现），Badge 在初始状态不显示。
- 「设置」页面的原有功能保持不变，仅入口位置从左侧菜单移至顶部栏。
- 顶部栏设置入口以图标（齿轮）形式呈现，图标旁可有文字「设置」也可无。
- 页面切换状态通过 URL hash（如 `#tasks`）实现，与现有路由机制保持一致。
