# Feature Specification: 全局文案中文化与 UI 规范统一

**Feature Branch**: `feat/010-i18n-ui-polish`

**Created**: 2026-05-25

**Status**: Draft

**Input**: 全局文案中文化和 UI 规范统一，不改功能逻辑。F04 — 全局英文文案清零；F05 — 全局 UI 规范统一。

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 全应用无英文用户可见文案 (Priority: P1)

用户使用 Mira 的全部五个页面时，无论进行何种操作，界面上所有可见文字均为中文。包括：页面标题、按钮文字、输入框提示、空状态说明、操作反馈提示、弹窗文案、错误提示。

**Why this priority**: 英文文案出现在用户界面中是产品定义的 Bug（见宪法 Principle VI），影响产品品质和用户信任感，需优先清零。

**Independent Test**: 逐一打开五个页面并触发所有交互状态（空态、加载态、成功、失败、确认弹窗），目视检查页面上所有可见文字，确认无英文出现。

**Acceptance Scenarios**:

1. **Given** 用户打开待办页面且列表为空，**When** 查看页面，**Then** 空态文案为中文「今天还没有待办，记录第一件事吧」，输入框提示为「记录一件事…」，无任何英文文案。
2. **Given** 用户打开笔记页面，**When** 查看各交互状态（空态、编辑中、已保存、删除确认），**Then** 所有文案均为中文，包括「+ 新建笔记」「开始记录…」「已自动保存」「确认删除这条笔记？」。
3. **Given** 用户打开问 Mira 页面，**When** 触发空态、加载态、复制操作，**Then** 文案分别显示中文欢迎语、「Mira 正在思考…」、「复制」/「已复制」。
4. **Given** 用户打开 Mira 的问题页面，**When** 有待处理和已处理的问题时，**Then** 标签分别显示「待回复」「已完成」，操作按钮显示「确认」「跳过」。
5. **Given** 用户打开统计页面，**When** 查看图表，**Then** 图表标题、坐标轴标签、图例说明均为中文。
6. **Given** 用户打开设置页面，**When** 查看各分组和字段，**Then** 所有 Tab 名称、字段标签、按钮文案均为中文。
7. **Given** 用户触发任意 Ant Design 组件的默认交互（Table 空态、DatePicker 日历、Modal 确认弹窗），**When** 组件渲染，**Then** 所有组件内置文案为中文（「暂无数据」「确认」「取消」而非英文默认值）。

---

### User Story 2 — 全应用布局间距规范统一 (Priority: P2)

用户在所有五个页面内浏览时，感受到一致的页面内边距、内容最大宽度约束和卡片视觉风格，不同页面之间没有明显的布局松散或拥挤差异。

**Why this priority**: 布局一致性是视觉品质的基础，先于美化细节。

**Independent Test**: 在不同页面间切换，目视测量内边距是否一致，卡片圆角和阴影是否统一；在宽屏下验证内容不超过最大宽度且居中显示。

**Acceptance Scenarios**:

1. **Given** 用户打开任意一个主页面，**When** 查看内容区域与窗口边缘的间距，**Then** 内容区域四周均有 24px 内边距，且在不同页面间保持一致。
2. **Given** 用户将 Mira 窗口拉宽（模拟大屏），**When** 内容宽度超过 960px，**Then** 内容自动居中对齐，两侧出现空白，不因拉宽而无限铺满。
3. **Given** 用户在任意页面查看卡片类容器（任务卡、笔记卡、设置分组），**When** 观察卡片样式，**Then** 统一显示 8px 圆角、浅灰色边框、轻微阴影，各页面视觉风格一致。
4. **Given** 用户查看任意页面内的列表项，**When** 观察相邻列表项之间的间距，**Then** 间距约为 12px，视觉上整齐均匀。
5. **Given** 用户查看任意页面内不同区块（Section）之间的间距，**When** 对比各页面，**Then** Section 间距约为 32px，与区块内间距有明显层次差异。

### User Story 3 — 全应用品牌色和控件规范统一 (Priority: P3)

用户在使用 Mira 时，主操作按钮、导航栏选中状态、重要数字、高亮标记均使用一致的品牌配色，输入框和按钮尺寸感一致，危险操作按钮有明确的红色视觉区分。

**Why this priority**: 品牌色一致性传递产品专业感，并通过颜色语义帮助用户区分主要操作、强调信息和危险操作。

**Independent Test**: 逐页检查主操作按钮颜色、导航栏选中色、强调数字色；点击触发危险操作（删除、重置），确认按钮为红色。

**Acceptance Scenarios**:

1. **Given** 用户查看应用导航侧栏，**When** 某页面处于选中状态，**Then** 该导航项背景色为品牌深蓝（`#1B2A4E`），文字为白色；未选中项 hover 时背景色变为浅灰（`#F5F5F5`）；导航栏固定宽度 200px。
2. **Given** 用户查看任意页面中的主操作按钮（如「发送」「保存设置」「新建笔记」），**When** 按钮处于正常状态，**Then** 背景色为品牌深蓝（`#1B2A4E`），鼠标悬停时颜色加深；按钮高度统一约 36px。
3. **Given** 用户查看任意页面中的统计数字、Badge 或重要高亮内容，**When** 该内容需要突出显示，**Then** 使用品牌金色（`#E8B86D`）作为高亮色，与主文本颜色形成视觉层次。
4. **Given** 用户触发危险操作（删除任务、删除笔记、重置数据），**When** 看到对应按钮或确认按钮，**Then** 按钮颜色为红色，与主操作按钮有明显视觉区分。
5. **Given** 用户查看任意输入框或表单控件，**When** 控件处于不可用（禁用）状态，**Then** 统一显示灰色，视觉一致，不出现不同页面使用不同禁用样式的情况。
6. **Given** 用户查看任意页面背景，**When** 页面处于正常浏览状态，**Then** 应用主背景色为浅灰（`#F8F9FA`），卡片内容区为白色（`#FFFFFF`），形成轻微的层次感。

---

### Edge Cases

- 统计页面图表库有自己的内置文案（如 tooltip、legend）：这些文案也需要是中文，或在图表配置中手动指定中文标签。
- 某些 Ant Design 组件（如 Upload、Pagination）在本次迭代中可能未用到：仅针对应用中实际渲染的组件验证中文化，未使用的组件不属于本次范围。
- 页面布局在应用窗口小于最大宽度时（960px 以下）：内容应填满宽度，24px 内边距保留，不出现水平滚动条。
- 品牌色应用于第三方图表组件：如图表库内置颜色无法通过全局样式覆盖，接受使用图表库默认调色板；强调色（金色）主要用于自定义的数字和 Badge。

## Requirements *(mandatory)*

### Functional Requirements

**F04 — 全局文案中文化**

- **FR-001**: 待办页面 MUST 在所有状态下使用中文文案：输入框提示「记录一件事…」、空态「今天还没有待办，记录第一件事吧」、完成分组「已完成 (N)」、删除确认「确认删除？」。
- **FR-002**: 笔记页面 MUST 在所有状态下使用中文文案：页面标题「笔记」、新建按钮「+ 新建笔记」、编辑器提示「开始记录…」、空态「还没有笔记，记录今天的第一个想法吧」、自动保存提示「已自动保存」、删除确认「确认删除这条笔记？」。
- **FR-003**: 问 Mira 页面 MUST 在所有状态下使用中文文案：页面标题「问 Mira」、总结区域标题「生成总结」、三个总结按钮「写日报」「写周报」「写月报」、输入框提示「问 Mira 任何工作相关的问题…」、发送按钮「发送」、加载提示「Mira 正在思考…」、复制按钮「复制」/「已复制」、空态中文欢迎语。
- **FR-004**: Mira 的问题页面 MUST 在所有状态下使用中文文案：页面标题「Mira 的问题」、空态「目前没有待确认的问题，Mira 会在需要时联系你」、状态标签「待回复」「已完成」、操作按钮「确认」「跳过」、完成提示「本次知识库更新完成」。
- **FR-005**: 统计页面 MUST 在所有图表元素上使用中文：页面标题「统计」、所有图表标题、坐标轴标签和图例。
- **FR-006**: 设置页面 MUST 对所有 Tab 名称、字段标签、按钮文案和状态提示使用中文。
- **FR-007**: 应用根组件 MUST 配置 Ant Design 全局中文语言包，覆盖所有 Ant Design 组件的内置文案，包括：Table 空态显示「暂无数据」（非「No Data」）、DatePicker 月份和星期名称为中文、Modal 确认弹窗按钮显示「确认」「取消」。
- **FR-008**: 所有通知提示（Toast / 消息）MUST 使用中文。
- **FR-009**: 所有错误提示文案 MUST 使用中文。

**F05 — 全局 UI 规范统一**

- **FR-010**: 所有五个主页面的内容区域 MUST 应用统一的 24px 四周内边距。
- **FR-011**: 所有页面内容 MUST 有最大宽度约束（960px），超出部分水平居中显示。
- **FR-012**: 所有卡片类容器 MUST 使用统一样式：8px 圆角、1px 浅灰边框（`#E8E8E8`）、轻微阴影。
- **FR-013**: 页面内元素间距 MUST 使用 8px 的倍数（8/16/24/32px），列表项间距约 12px，Section 间距约 32px。
- **FR-014**: 所有输入框和按钮 MUST 统一高度约 36px（中等尺寸）。
- **FR-015**: 主操作按钮 MUST 使用品牌深蓝背景（`#1B2A4E`），鼠标悬停时颜色加深；危险操作按钮 MUST 使用红色；禁用状态 MUST 统一为灰色。
- **FR-016**: 导航侧栏 MUST 固定宽度 200px；选中项背景色 MUST 为 `#1B2A4E`（白色文字）；未选中项 hover 背景色 MUST 为 `#F5F5F5`。
- **FR-017**: 重要数字、Badge、高亮内容 MUST 使用品牌金色（`#E8B86D`）作为强调色。
- **FR-018**: 应用主背景色 MUST 为浅灰（`#F8F9FA`），卡片内容区背景 MUST 为白色（`#FFFFFF`）。
- **FR-019**: 上述所有样式改动 MUST 不改变任何现有功能逻辑，不影响数据的增删查改交互行为。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 逐页视觉审查后，五个主页面及所有交互状态（空态、加载态、成功、失败、弹窗）中的英文可见文案数量为 0。
- **SC-002**: 随机抽取至少 3 个 Ant Design 组件（如 Table、DatePicker、Modal）进行操作验证，所有内置文案均显示中文，无英文残留。
- **SC-003**: 在五个页面各随机测量 3 处内边距，所有测量值均为 24px，误差不超过 1px。
- **SC-004**: 卡片圆角视觉验证通过（8px），不同页面间卡片风格无肉眼可见差异。
- **SC-005**: 主操作按钮颜色与设计规范一致（品牌深蓝），危险按钮为红色，两者不混用。
- **SC-006**: 所有功能性测试（增删改查、AI 请求、设置保存）在样式改动后行为无变化，无功能性回归。

## Assumptions

- 本次规范统一以「先对齐，后美化」为原则：当现有布局与规范有冲突时，优先保证布局不破坏功能，其次对齐规范值。
- 第三方图表组件（如统计页使用的图表库）若内置文案不支持简单配置中文，则接受通过图表配置项手动指定中文标签，不强求通过全局覆盖解决。
- 品牌金色（`#E8B86D`）的强调色使用场景由实现者判断，主要用于已有「重要数字」或「高亮」意图的现有元素，不新增强调色装饰。
- 每个页面的样式改动完成后需独立验证（不批量改动后统一验证），以便快速发现回归。
- 不涉及响应式多端设计：Mira 为桌面应用，只需保证在常见桌面窗口尺寸下布局正确。
