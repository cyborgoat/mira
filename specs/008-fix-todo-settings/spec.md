# Feature Specification: 修复待办完成标记与 AI 设置功能

**Feature Branch**: `fix/008-fix-todo-settings`

**Created**: 2026-05-25

**Status**: Draft

**Input**: User description: 修复现有功能缺失问题，不新增页面，只修复已有页面。F01 — Todo 页面完成标记；F02 — 设置页大模型配置还原并真实可调。

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 标记待办任务为已完成 (Priority: P1)

用户在待办页面看到任务列表，每条任务左侧有可点击的圆形完成标记。点击后，该任务移入页面底部的「已完成」区域，视觉上变灰并带删除线，清晰与未完成任务区分。

**Why this priority**: 完成标记是任务管理的核心交互，缺失此功能会导致整个 Todo 页面失去实用价值。

**Independent Test**: 进入待办页面，点击任意一条任务的完成标记，验证该任务移至「已完成」区域、文字带删除线且颜色变灰。

**Acceptance Scenarios**:

1. **Given** 待办页面有若干未完成任务，**When** 用户点击任意任务左侧的圆形完成标记，**Then** 该任务立即移入页面底部「已完成」区域，文字显示删除线且颜色变为灰色。
2. **Given** 「已完成」区域为空，**When** 用户完成第一条任务，**Then** 「已完成」区域出现，标题显示「已完成 (1)」，默认折叠状态。
3. **Given** 「已完成」区域有 N 条任务，**When** 用户再次完成一条任务，**Then** 标题中的计数更新为「已完成 (N+1)」。
4. **Given** 用户刷新或重启应用，**When** 再次进入待办页面，**Then** 已完成的任务仍保留在「已完成」区域（完成状态持久化）。

---

### User Story 2 — 管理已完成任务（撤销完成与删除）(Priority: P2)

用户可展开「已完成」区域查看所有已完成任务，支持将任务撤销为待办，或永久删除已完成任务。

**Why this priority**: 撤销和删除是完成标记功能的完整性补充，用户可能误操作或需要清理列表。

**Independent Test**: 完成一条任务后，进入「已完成」区域，点击撤销，验证任务回到待办列表；再次完成，点击删除，验证任务从列表消失。

**Acceptance Scenarios**:

1. **Given** 「已完成」区域折叠，**When** 用户点击「已完成 (N)」标题，**Then** 区域展开，显示所有已完成任务；再次点击则折叠。
2. **Given** 「已完成」区域已展开，**When** 用户点击某已完成任务的圆形标记（撤销操作），**Then** 该任务恢复为未完成状态并移回待办列表，删除线和灰色样式消失。
3. **Given** 「已完成」区域已展开，**When** 用户点击某已完成任务的删除操作，**Then** 该任务从列表永久移除，计数随之减少。
4. **Given** 所有已完成任务被撤销或删除，**When** 「已完成」区域变为空，**Then** 区域及其标题自动隐藏。

---

### User Story 3 — 配置 AI 设置并验证连接 (Priority: P3)

用户在设置页面可以完整填写 AI 服务配置（API Key、模型名称、接口地址），点击「测试连接」验证配置是否有效，保存后所有 AI 功能正常运行。

**Why this priority**: AI 功能（Ask Mira、Copilot、Lint）必须依赖正确配置的 API Key 才能运行，设置页面的配置功能不可用会导致 AI 相关功能全部失效。

**Independent Test**: 进入设置页面，填写有效 API Key，点击「测试连接」，验证显示绿色「连接成功」；填写无效 Key，验证显示红色「连接失败」及错误原因。

**Acceptance Scenarios**:

1. **Given** 用户进入设置页面，**When** 查看「AI 设置」面板，**Then** 面板包含三个字段：API Key（密码框）、模型名称（默认 claude-haiku-4-5）、接口地址（默认 https://api.anthropic.com）；API Key 字段有显示/隐藏切换按钮。
2. **Given** 用户填写有效 API Key 并点击「测试连接」，**When** 测试请求成功，**Then** 按钮附近显示绿色「连接成功」以及模型返回的第一句话。
3. **Given** 用户填写无效 API Key 并点击「测试连接」，**When** 测试请求失败，**Then** 显示红色「连接失败：[具体错误原因]」。
4. **Given** 用户填写完整配置并点击「保存设置」，**When** 保存成功，**Then** 此后 Ask Mira、Copilot 建议、Lint 功能均可正常调用 AI。
5. **Given** 用户点击「清除 Key」，**When** 确认操作，**Then** API Key 字段清空，AI 功能进入「未配置」状态并提示用户前往设置配置 Key。

---

### Edge Cases

- 用户点击「测试连接」时，接口地址填写了格式错误的 URL：应显示「连接失败：接口地址格式无效」。
- 网络断开时点击「测试连接」：应显示「连接失败：网络不可达」。
- 「测试连接」请求超时（超过 10 秒）：应中断请求并显示「连接失败：请求超时」。
- 「已完成」区域为空时，该区域及标题不应显示（不显示空的折叠区域）。
- 应用关闭后重新启动，「已完成」区域的折叠/展开状态无需保留（默认折叠即可）。

## Requirements *(mandatory)*

### Functional Requirements

**F01 — 待办完成标记**

- **FR-001**: 待办列表每条任务 MUST 在左侧显示圆形完成标记（Checkbox），未完成时为空心圆。
- **FR-002**: 用户点击未完成任务的完成标记后，系统 MUST 将该任务状态更新为已完成，并将其移入「已完成」区域。
- **FR-003**: 已完成任务 MUST 以删除线和灰色文字区别于未完成任务。
- **FR-004**: 「已完成」区域 MUST 显示在待办列表底部，标题格式为「已完成 (N)」，N 为当前已完成任务数量。
- **FR-005**: 「已完成」区域 MUST 支持点击标题折叠/展开，默认为折叠状态。
- **FR-006**: 用户 MUST 能够点击已完成任务的完成标记，将其撤销为待办状态，恢复到待办列表。
- **FR-007**: 用户 MUST 能够删除已完成任务，删除操作应仅对已完成区域的任务可用。
- **FR-008**: 任务的完成状态 MUST 在应用重启后保留（持久化存储）。
- **FR-009**: 「已完成」区域在任务数为零时 MUST 自动隐藏。

**F02 — 设置页 AI 配置**

- **FR-010**: 设置页面 MUST 提供完整的「AI 设置」面板，包含 API Key、模型名称、接口地址三个输入字段。
- **FR-011**: API Key 字段 MUST 默认以密码形式显示（内容不可读），并提供显示/隐藏切换按钮。
- **FR-012**: 模型名称字段 MUST 有默认值 `claude-haiku-4-5`，用户可修改。
- **FR-013**: 接口地址字段 MUST 有默认值（标准 AI 服务接口地址），用户可修改以支持自定义兼容接口。
- **FR-014**: 「AI 设置」面板 MUST 提供「测试连接」按钮，点击后向已配置接口发送一条最小测试请求。
- **FR-015**: 测试连接成功时，系统 MUST 在按钮旁显示绿色中文成功提示及模型返回内容的第一句话。
- **FR-016**: 测试连接失败时，系统 MUST 在按钮旁显示红色中文失败提示及具体错误原因。
- **FR-017**: 「测试连接」请求 MUST 在 10 秒内完成，超时应中止并以失败状态呈现。
- **FR-018**: 用户点击「保存设置」后，所有依赖 AI 的功能（Ask Mira、Copilot、Lint）MUST 使用更新后的配置进行后续调用。
- **FR-019**: 面板 MUST 提供「清除 Key」功能，清除后 API Key 字段置空，AI 功能进入未配置状态。
- **FR-020**: 所有按钮文案和状态提示 MUST 使用中文。

### Key Entities

- **Task（任务）**: 待办事项，具有标题、完成状态（已完成/未完成）等属性。完成状态是核心变更字段。
- **LlmSettings（AI 设置）**: 存储 API Key（加密存储）、模型名称、接口地址，控制所有 AI 功能的可用性。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户完成标记任务的操作（点击 → 移入「已完成」区域）在 300 毫秒内完成，无感知延迟。
- **SC-002**: 任务完成状态在应用重启后 100% 保留，不丢失。
- **SC-003**: 填写有效配置后点击「测试连接」，5 秒内显示明确的成功/失败结果，用户无需等待超过 10 秒。
- **SC-004**: 保存有效 AI 设置后，Ask Mira 页面能够成功收到 AI 回复（端到端验证）。
- **SC-005**: 所有新增/修改的用户可见文案 100% 为中文，无英文残留。
- **SC-006**: 修复范围严格限定在待办页和设置页，其他页面功能测试无回归。

## Assumptions

- 任务数据存储层（Rust 侧）已存在 `completed` 字段，前端绑定缺失是问题所在；本 spec 不涉及数据库 schema 变更。
- AI 设置中的 API Key 在存储和传输环节仍由系统安全层负责（遵循宪法 Principle VIII），本 spec 关注的是用户交互层面。
- 「测试连接」只需验证 API Key 和接口地址的连通性（发送「你好」），不需要验证模型名称的合法性。
- 删除已完成任务不需要二次确认弹窗（直接删除），与待办列表的现有删除行为保持一致。
- 本次修复不调整任何其他页面（笔记页、Ask Mira、Mira Ask、统计页）的功能逻辑。
