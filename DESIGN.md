# Mira Design Notes

Mira is a product UI, so design serves repeated work. Prefer restrained contrast, compact controls, clear hierarchy, and familiar interaction patterns.

Guidelines:
- Keep primary workflows visible without large marketing sections.
- Use existing Mira components and spacing before adding new visual patterns.
- Use cards only for concrete panels or repeated items.
- Favor short labels, clear empty states, and visible loading/error feedback.
- Markdown reading should be plain, legible, and safe for daily browsing.
